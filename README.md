# 별보리 (Byeol-Bori)

## 프로젝트 개요

교회 청년부 수련회에서 실제 사용을 목적으로 제작한 실시간 체험형 스토리 게임 시스템

- 제작 목적: 실제 행사 운영
- 제작 기간: 1개월
- 제작 비용: 0원
- 운영 규모: 10명 내외 동시 참여

푸시 알림 기반 진행 제어와  
다수 참가자 동시 참여 환경 고려 구조

---

## 프로젝트 성격

- 실제 행사 운영 목적 시스템
- 현장 진행 흐름 중심 구조
- 1인 기획 프로젝트
- 구현 단계에서 AI 도구 활용

---

## 기획 관점 정리

- 디자인 실무 경험 기반 화면 흐름 판단
- 제약 조건 전제 하 구조 설계 경험
- 구현 자체보다 운영 가능성 중심 판단
- AI를 구현 수단으로 활용, 의사결정은 기획자 책임

---

## 사용 환경

- 오프라인 행사 현장
- 참가자 개인 모바일 기기 사용
- 관리자 1인 전체 진행 제어
- 전원 동시 참여 방식

---

## 역할 구분

기획자와 AI 도구 역할 분리 구조

기획자 (본인)
- 서비스 전체 구조 설계
- 사용자 흐름 정의
- 인터랙션 규칙 정의
- 퍼즐 유형 구성
- 난이도 조정 기준 설정
- 운영 정책 및 시나리오 수립
- 현장 사용 결과 확인 및 수정 판단

AI 도구
- 정의된 설계 기반 코드 구현
- 기획자 요청 사항 반영

---

## 핵심 구조

### 엔진–콘텐츠 분리 구조

- 퍼즐 로직과 콘텐츠 데이터 분리
- 콘텐츠 교체 중심 운영 구조
- 동일 구조 기반 다수 미션 운영

구성 요소
- 6종 인터랙션 엔진
- 미션 단위 콘텐츠 구성
- 운영 중 콘텐츠 교체 가능 구조

---

### 콘텐츠 관리 방식

좌표 기반 퍼즐 일부에 한해 적용

- 화면 기반 좌표 지정 방식
- 수기 좌표 입력 방식 배제

결과
- 해당 퍼즐에서 좌표 입력 오류 발생 없음
---

### 실시간 운영 대응

- 관리자 중심 단계 전환 구조
- 참가자 진행 상태 동기화
- 재접속 및 새로고침 대응
- 현장 네트워크 불안정 상황 고려

---

## 구현 환경

※ 기술 역량 설명 목적 아님  
※ 실제 사용을 위한 최소 구현 환경

- Frontend: 웹 표준 기술
- Backend: PHP (개인 서버)
- Push: 외부 푸시 서비스 활용
- Storage: 브라우저 저장소 + JSON

---

## 주요 기능

사용자 영역
- 6종 인터랙션 유형
  - 순차 터치
  - 정답 선택
  - 조각 맞추기
  - 선 연결
  - 경로 따라가기
  - 액체 이동 퍼즐
- 오답 누적 시 미션 자동 교체
- 진행 상태 자동 저장 및 복구
- 힌트 중복 차감 방지
- 개인 메모 저장 및 불러오기

관리자 영역
- 푸시 알림 기반 단계 전환
- 관리자 기기 기준 진행 데이터 초기화
- 개별 미션 테스트 환경
